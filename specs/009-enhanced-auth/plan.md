# Implementation Plan: Enhanced Authentication

**Branch**: `009-enhanced-auth` | **Date**: 2026-02-22 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/009-enhanced-auth/spec.md`

## Summary

Add four authentication capabilities to the voter-api: (1) self-service password reset via emailed token, (2) admin-initiated user invitations, (3) TOTP two-factor authentication with recovery codes, and (4) passkey (WebAuthn) registration and login as a standalone alternative to password login.

Email delivery uses the official Mailgun Python SDK (`mailgun`) with its `AsyncClient`. TOTP uses `pyotp` with `segno` for QR code generation. TOTP secrets are encrypted at rest with Fernet (`cryptography`). Passkeys use `py-webauthn` with stateless JWT challenge tokens. The existing login endpoint migrates from OAuth2 form-data to JSON body and gains an optional `totp_code` field.

All new capabilities are implemented as standalone libraries in `src/voter_api/lib/` before integration into services and routes, per the Library-First architecture principle.

## Technical Context

**Language/Version**: Python 3.13 (see `.python-version`)
**Primary Dependencies**: FastAPI, SQLAlchemy 2.x async, Pydantic v2; pyotp, segno, cryptography, py-webauthn, mailgun (new)
**Storage**: PostgreSQL 15+ / PostGIS — 4 new tables across 4 Alembic migrations (025–028)
**Testing**: pytest with pytest-asyncio, pytest-cov; 90% coverage threshold enforced
**Target Platform**: Linux server (piku deploy), asyncpg driver
**Project Type**: API-only (no frontend); passkey ceremonies initiated by external clients
**Performance Goals**: Auth endpoints < 500ms p95; email delivery is synchronous (fail-fast); passkey challenge tokens expire in 5 minutes
**Constraints**: TOTP secrets encrypted at rest; token/code hashes stored (not plaintext); rate-limit password reset to 1 per email per 5 minutes; lock TOTP after 5 consecutive failures for 15 minutes

## Implementation Phases

### Phase 1 — Foundation: Libraries, Models, Migrations

**Goal**: All core logic as testable standalone libraries; database schema in place.

1. **Install new dependencies**:
   ```bash
   uv add pyotp segno cryptography py-webauthn mailgun
   ```

2. **Extend `core/config.py`** with Mailgun, TOTP, WebAuthn, and rate-limit settings.

3. **Extend `.env.example`** with all 11 new env vars (with placeholder values). See `quickstart.md` for the authoritative list.

4. **Implement `lib/mailer/`** (`MailgunMailer`):
   - `send_email(to, subject, html_body)` — async call via `mailgun.client.AsyncClient`
   - `render_template(name, context)` — Jinja2 template rendering
   - Raises `MailDeliveryError` (subclass of `RuntimeError`) on non-2xx response
   - Jinja2 templates: `invite.html`, `password_reset.html`
   - Unit tests: mock `mailgun.client.AsyncClient`, assert correct payload and error propagation

5. **Implement `lib/totp/`** (`TOTPManager`):
   - `generate_secret()` → encrypted base32 secret (Fernet)
   - `get_provisioning_uri(encrypted_secret, username, issuer)` → `otpauth://` URI
   - `get_qr_svg(provisioning_uri)` → inline SVG string (segno)
   - `verify_code(encrypted_secret, code)` → bool (with 30s window tolerance)
   - `generate_recovery_codes(n=10)` → list of raw codes + list of their SHA-256 hashes
   - `verify_recovery_code(raw_code, stored_hashes)` → bool
   - Unit tests: code generation, verification, replay rejection, recovery code roundtrip

6. **Implement `lib/passkey/`** (`PasskeyManager`):
   - `generate_registration_options(user_id, username, existing_credentials)` → options dict + raw challenge bytes
   - `verify_registration(credential_response, expected_challenge, expected_origin, expected_rp_id)` → verified credential
   - `generate_authentication_options(credentials)` → options dict + raw challenge bytes
   - `verify_authentication(credential_response, expected_challenge, credential_public_key, sign_count, expected_origin, expected_rp_id)` → new sign_count
   - Unit tests: mocked py_webauthn; verify correct argument passing and error propagation

7. **Add new models** (`auth_tokens.py`, `totp.py`, `passkey.py`), extend `models/__init__.py`.

8. **Write Alembic migrations 025–028**.

9. **Commit**: `feat(auth): add auth library layer and DB migrations (025-028)`

### Phase 3 — TOTP Two-Factor Authentication

**Goal**: P3 story fully functional; TOTP-enabled accounts require code at login.

1. **Extend `schemas/auth.py`** with:
   - `TOTPEnrollmentResponse`, `TOTPConfirmRequest`, `TOTPConfirmResponse`, `TOTPRecoveryCodesCountResponse`
   - Extend `LoginRequest` with `totp_code: str | None = None`
   - Add `MFARequiredError` response schema

2. **Modify `auth_service.authenticate_user`** to:
   - After password verification: check if `totp_credential.is_verified`
   - If TOTP active and `totp_code` is None → raise `MFARequiredError`
   - If TOTP active and `totp_code` provided → verify via `TOTPManager`; check lockout; update `failed_attempts`
   - If code is recovery-code length (>6 chars) → verify against `totp_recovery_codes`; mark used; check lockout bypass

3. **Add to `auth_service.py`**:
   - `enroll_totp(session, totp_manager, user)` → create/replace pending `TOTPCredential`; return URI + QR
   - `confirm_totp(session, totp_manager, user, code)` → verify code → mark `is_verified=true`; generate/store 10 recovery code hashes; return raw codes
   - `disable_totp(session, user_id)` → delete `TOTPCredential` + all `TOTPRecoveryCode` rows
   - `unlock_totp(session, user_id)` → clear `locked_until` and `failed_attempts`
   - `get_recovery_code_count(session, user_id)` → count unused rows

4. **Extend `api/v1/auth.py`** with:
   - `POST /auth/totp/enroll`
   - `POST /auth/totp/confirm`
   - `DELETE /auth/totp`
   - `GET /auth/totp/recovery-codes/count`
   - `DELETE /users/{id}/totp` (admin)
   - `POST /users/{id}/totp/unlock` (admin)

5. **Modify `POST /auth/login`**: change from `OAuth2PasswordRequestForm` to JSON `LoginRequest`; propagate TOTP error as 403 `MFARequiredError`; propagate lockout as 429 `TOTPLockedError`.

6. **Integration tests**: TOTP enrollment → confirm → login with/without code; lockout at 5 failures; recovery code bypasses lockout.

7. **Extend E2E** with TOTP-enabled user fixture and `TestAuth` TOTP smoke tests.

8. **Lint + test**.

9. **Commit**: `feat(auth): add TOTP two-factor authentication`

### Phase 4 — Passkey Login

**Goal**: P4 story fully functional; passkeys work as standalone login alternative.

1. **Extend `schemas/auth.py`** with:
   - `PasskeyRegistrationOptionsResponse`, `PasskeyRegistrationVerifyRequest`, `PasskeyResponse`, `PasskeyRenameRequest`
   - `PasskeyLoginOptionsRequest`, `PasskeyLoginOptionsResponse`, `PasskeyLoginVerifyRequest`

2. **Add to `auth_service.py`**:
   - `get_passkey_registration_options(session, passkey_manager, settings, user)` → options + encode challenge JWT
   - `verify_passkey_registration(session, passkey_manager, settings, user, credential_response, challenge_token, name)` → create `Passkey` row
   - `list_passkeys(session, user_id)` → list of `Passkey` rows
   - `rename_passkey(session, user_id, passkey_id, name)` → update name
   - `delete_passkey(session, user_id, passkey_id)` → delete row
   - `get_passkey_login_options(session, passkey_manager, settings, username)` → options + challenge JWT
   - `verify_passkey_login(session, passkey_manager, settings, username, credential_response, challenge_token)` → verify assertion → update `sign_count` + `last_used_at` → return `TokenResponse`

3. **Challenge JWT helpers** in `core/security.py`:
   - `create_passkey_challenge_token(username, challenge_b64, settings)` → signed JWT (5-min exp, type=`passkey_challenge`)
   - `decode_passkey_challenge_token(token, settings)` → `{"username", "challenge_b64"}`

4. **Extend `api/v1/auth.py`** with all 7 passkey endpoints.

5. **Integration tests**: registration options → verify → list → rename → delete; login options → verify → tokens issued.

6. **Extend E2E** with passkey registration and login smoke tests (using py_webauthn test helpers / mock authenticator).

7. **Lint + test**.

8. **Commit**: `feat(auth): add passkey (WebAuthn) registration and login`
