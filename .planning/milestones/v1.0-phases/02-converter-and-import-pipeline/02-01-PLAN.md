<objective>
Create the Alembic migration and update all ORM models, API schemas, and E2E tests for the candidate-to-candidacy refactoring and ElectionEvent enhancement. This unblocks Plan 02-03 (import pipeline) which needs the new DB structure to import JSONL records.

Purpose: The existing Candidate model ties a person to a single election. The new model separates the person (Candidate) from their participation in an election (Candidacy junction table). ElectionEvent also needs calendar/feed fields that are currently on the Election model. These structural changes must land first so the import pipeline can target the correct DB schema.

Output: Updated ORM models, new Candidacy model, Alembic migration with data migration, updated API schemas and routes, all E2E tests passing.
</objective>

         ```sql
         INSERT INTO candidacies (id, candidate_id, election_id, party, filing_status,
           ballot_order, is_incumbent, contest_name, qualified_date, occupation,
           home_county, municipality, sos_ballot_option_id, import_job_id, created_at, updated_at)
         SELECT gen_random_uuid(), c.id, c.election_id, c.party, c.filing_status,
           c.ballot_order, c.is_incumbent, c.contest_name, c.qualified_date, c.occupation,
           c.home_county, c.municipality, c.sos_ballot_option_id, c.import_job_id, now(), now()
         FROM candidates c
         WHERE c.election_id IS NOT NULL
         ON CONFLICT (candidate_id, election_id) DO UPDATE
           SET party = EXCLUDED.party,
               filing_status = EXCLUDED.filing_status,
               ballot_order = EXCLUDED.ballot_order,
               is_incumbent = EXCLUDED.is_incumbent,
               contest_name = EXCLUDED.contest_name,

               municipality = EXCLUDED.municipality,
               sos_ballot_option_id = EXCLUDED.sos_ballot_option_id,
               import_job_id = EXCLUDED.import_job_id,
               updated_at = now()
         ```
      5. Copy calendar fields from elections to election_events where election_event_id is set
    - The downgrade should: drop candidacies table, remove new columns, backfill any NULL election_id values (e.g., DELETE FROM candidates WHERE election_id IS NULL), then make election_id NOT NULL again
