<objective>
Build the deterministic markdown-to-JSONL converter library, its CLI command, and populate all 159 county reference files with governing body data. This is the core novel component of Phase 2 -- everything else follows established patterns.

Purpose: The converter is the bridge between human-authored/reviewed markdown files and the machine-readable JSONL that feeds the database import pipeline. It must be deterministic (no AI), validate against Phase 1 JSONL schemas, and resolve Body/Seat references to boundary_type using county reference files.

Output: Working `lib/converter/` library with unit tests, `voter-api convert` CLI command, all 159 county reference files populated with Governing Bodies tables.
</objective>

  <behavior>
    Parser (test_parser.py):
    - Parses election overview markdown -> extracts metadata table (ID, Format Version, Type, Stage, election dates, calendar fields)
    - Parses single-contest markdown -> extracts metadata + candidate table (Name, Party, Filing Status, Ballot Order, Incumbent)
    - Parses multi-contest markdown -> extracts metadata + per-contest sections (### headings) with Body/Seat metadata lines and candidate tables
    - Extracts Body/Seat from per-contest metadata line format: **Body:** {id} | **Seat:** {id}
    - Returns ParseResult dataclass with file_type, metadata dict, contests list, errors list
    - Missing/invalid UUID -> validation error in ParseResult.errors
    - Handles 5-column and 7-column candidate tables (legacy format has 7 columns)

    Writer (test_writer.py):
    - Converts ParseResult for overview -> ElectionEventJSONL record (validates via model_validate)
    - Converts ParseResult for single-contest -> ElectionJSONL record
    - Converts ParseResult for multi-contest -> multiple ElectionJSONL records (one per ### contest section)
    - Writes validated records to JSONL file (one JSON object per line)
    - Invalid record (Pydantic validation failure) -> logged in ConversionReport.errors, record skipped

    Resolver (test_resolver.py):
    - Resolves statewide Body IDs (ga-governor, ga-us-senate, etc.) to boundary_type via built-in mapping
    - Resolves county Body IDs (bibb-boe, bibb-superior-court, etc.) to boundary_type via county reference file lookup
    - Unknown Body ID -> validation error
    - Missing county reference file -> validation error
    - Parses Governing Bodies table from county reference markdown to build {body_id: boundary_type} mapping

    Report (test_report.py):
    - ConversionReport tracks: files_processed, files_succeeded, files_failed, errors list, warnings list
    - Renders terminal table (human-readable summary)
    - Renders JSON report file (machine-readable, written to output directory)
    - Report includes per-file status with error details

    Directory (test_directory.py):
    - convert_directory() walks election directory, identifies file types by path pattern
    - Overview file at root (date-type.md pattern)
    - Single-contest files at root (date-contest.md pattern)
    - Multi-contest files in counties/ subdirectory
    - Output JSONL files written to sibling jsonl/ directory (or --output override)
    - Returns ConversionReport with aggregate results
  </behavior>
  <action>
    First install mistune: `uv add mistune`

    Create `src/voter_api/lib/converter/types.py`:
    - `ParseResult` dataclass: file_path, file_type (enum: overview, single_contest, multi_contest), metadata (dict), contests (list of ContestData), errors (list of str), warnings (list of str)
    - `ContestData` dataclass: heading, body_id, seat_id, candidates (list of dict), metadata (dict)
    - `ConversionResult` dataclass: file_path, records (list of dict), errors (list of str)

    Create `src/voter_api/lib/converter/parser.py`:
    - `parse_markdown(file_path: Path) -> ParseResult` -- main entry point
    - Uses `mistune.create_markdown(renderer=None, plugins=['table'])` to get AST tokens
    - `_extract_metadata_table(tokens)` -- extracts first `| Field | Value |` table as dict
    - `_extract_candidate_table(tokens)` -- extracts candidate data from table tokens (handles both 5 and 7 column formats)
    - `_extract_contests(tokens)` -- for multi-contest files, splits by ### headings, extracts Body/Seat from metadata line
    - `_validate_uuid(metadata)` -- checks ID field per uuid-strategy.md rules (missing, invalid, empty all -> error)
    - `_detect_file_type(file_path, metadata)` -- determines overview vs single-contest vs multi-contest

    Create `src/voter_api/lib/converter/resolver.py`:
    - `STATEWIDE_BODIES: dict[str, str]` -- built-in mapping of statewide/federal Body IDs to boundary_type values. Include: ga-governor, ga-lt-governor, ga-sos, ga-ag, ga-insurance-commissioner, ga-labor-commissioner, ga-school-superintendent, ga-psc (public service commission), ga-us-senate, ga-us-house (congressional), ga-state-senate (state_senate), ga-state-house (state_house), ga-supreme-court, ga-court-of-appeals, ga-superior-court (judicial_circuit). Exact values from boundary-types.md vocabulary.
    - `load_county_references(counties_dir: Path) -> dict[str, dict[str, str]]` -- loads all county reference files, returns {county_slug: {body_id: boundary_type}}
    - `resolve_body(body_id: str, county_refs: dict) -> str | None` -- resolves Body ID to boundary_type. Checks statewide first, then county references. Returns None if unresolved.
    - `parse_governing_bodies(file_path: Path) -> dict[str, str]` -- parses the Governing Bodies table from a county reference markdown file. Table format: `| Body Name | Body ID | Boundary Type | Election Type | Seats |`

    Create `src/voter_api/lib/converter/writer.py`:
    - `write_jsonl(records: list[dict], output_path: Path, model_class: type) -> tuple[int, list[str]]` -- validates each record against Pydantic model, writes valid records to JSONL file, returns (count_written, errors)
    - `parse_result_to_records(result: ParseResult, resolver: callable) -> list[ConversionResult]` -- converts ParseResult to JSONL-ready dicts by mapping metadata fields to JSONL schema fields and resolving Body IDs to boundary_type

    Create `src/voter_api/lib/converter/report.py`:
    - `ConversionReport` class with methods: `add_success(file_path, record_count)`, `add_failure(file_path, errors)`, `add_warning(file_path, message)`, `render_terminal() -> str`, `write_json(output_path: Path)`
    - Terminal rendering: Rich table or plain text table showing file status, record counts, error summary
    - JSON output: `{"total_files": N, "succeeded": N, "failed": N, "files": [...], "errors": [...]}`

    Create `src/voter_api/lib/converter/__init__.py`:
    - Public API: `convert_directory(directory: Path, output: Path | None = None, fail_fast: bool = False) -> ConversionReport`
    - Public API: `convert_file(file_path: Path, output: Path | None = None) -> ConversionResult`
    - `__all__` listing public functions

    Write unit tests using Bibb election data as fixtures (create markdown fixture strings in test files matching the format specs). Test each module independently. Use `tmp_path` for file output.

    IMPORTANT: The converter has NO database dependency. It reads markdown files and writes JSONL files. All validation is against Pydantic models from schemas/jsonl/.

    IMPORTANT: Per uuid-strategy.md, the converter MUST NOT generate UUIDs. Missing/invalid UUID = validation error.

      ```markdown
      ## Governing Bodies

      | Body Name | Body ID | Boundary Type | Election Type | Seats |

      ```

      - **Board of Commissioners (BOC)**: Most counties have a BOC or equivalent county governing authority. Body ID: `{county}-boc`. Boundary type: `county_commission`.
      - **Superior Court**: All counties belong to a judicial circuit. Body ID: `{county}-superior-court`. Boundary type: `judicial_circuit`.
      - **State Court**: Not all counties have a state court. Only include if the county has one. Body ID: `{county}-state-court`. Boundary type: `county`.
      - **Probate Court**: All counties. Body ID: `{county}-probate-court`. Boundary type: `county`. Typically 1 judge (sole seat).
      - **Magistrate Court**: All counties. Body ID: `{county}-magistrate-court`. Boundary type: `county`. Typically 1 chief magistrate (sole seat).
      - **County Coroner** or **County Medical Examiner**: Most counties elect a coroner. Body ID: `{county}-coroner`. Boundary type: `county`. Sole seat.
      - **County Sheriff**: All counties. Body ID: `{county}-sheriff`. Boundary type: `county`. Sole seat.
      - **County Tax Commissioner**: Most counties. Body ID: `{county}-tax-commissioner`. Boundary type: `county`. Sole seat.
      - **County Clerk of Courts**: Most counties. Body ID: `{county}-clerk-superior-court`. Boundary type: `county`. Sole seat.
