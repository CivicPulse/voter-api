### Integration Issues

**1. election_event_id FK stored as NULL (Moderate — DEM-01)**

`election_import_service._prepare_record()` correctly detects the placeholder UUID (`00000000-...`) and skips it, storing NULL for `election_event_id`. The existing `voter-api import resolve-elections` command could populate this FK, but it is absent from the demo walkthrough. The walkthrough text at line 366 inaccurately states the FK "is resolved during import." Impact: elections-to-events join unpopulated; no effect on listing/search queries.

**2. May 19 has no candidate data (Informational — intentional)**

Only March 10 (39 candidates) and March 17 (10 candidates) have `candidates.jsonl` and `candidacies.jsonl`. May 19 has 25 elections but zero candidates. This is documented in the walkthrough (line 873) as future work when qualifying files are released.
