  <action>
    First, install hypothesis: `uv add --dev hypothesis`

    Create normalize.py with two public functions:

    1. normalize_file(file_path, *, dry_run=False) -> NormalizationResult:
       - Read file content
       - Detect file type (overview, single-contest, multi-contest, candidate) by inspecting metadata table and path
       - For election files: apply to all metadata table values: smart_title_case on name fields, normalize_date on date fields, normalize_url on URL fields
       - For candidate tables: apply smart_title_case to Candidate column, normalize_occupation to Occupation column, normalize_date to Qualified Date column
       - For candidate markdown files: apply smart_title_case to name fields, normalize_url to links, normalize_occupation to occupation
       - Validate structural consistency: required metadata fields present, section ordering correct
       - Flag ALL CAPS remnants (words of 4+ chars still in ALL CAPS after processing) as warnings
       - If not dry_run, write normalized content back
       - Return NormalizationResult with changes list

    2. normalize_directory(directory, *, dry_run=False, report_path=None) -> NormalizationReport:
       - Walk directory, find all .md files (skip README.md)
       - Call normalize_file for each
       - Build NormalizationReport from results
       - If report_path provided, write JSON report
       - Print terminal report

    Create uuid_handler.py:
    1. ensure_uuid(content: str) -> tuple[str, str | None]:
       - Check metadata table for ID field
       - If ID is empty or "--", generate uuid4, write into content, return (new_content, generated_uuid)
       - If ID is valid UUID, return (content, None)
       - If ID is invalid, raise ValueError

    2. rename_candidate_file(file_path: Path, uuid_str: str) -> Path | None:
       - If filename contains "00000000" placeholder, replace with first 8 chars of UUID
       - Rename file on disk, return new path
       - If no placeholder found, return None

    Update __init__.py to import normalize_directory, normalize_file from normalize.py and NormalizationReport from report.py. Update __all__.

    Create golden file fixtures:
    - tests/fixtures/normalizer/before/ -- 4 files with intentional formatting issues:
      - election-overview.md: ALL CAPS election name, unpadded dates, missing Format Version UUID
      - single-contest.md: ALL CAPS candidate names, http:// URLs, inconsistent date formats
      - multi-contest.md: ALL CAPS contest names, missing https://, occupation in ALL CAPS
      - candidate-file.md: ALL CAPS name, placeholder ID (00000000), http:// URLs, ALL CAPS occupation
    - tests/fixtures/normalizer/after/ -- 4 matching files with expected normalized output
    - Fixtures should exercise actual GA SOS data patterns from RESEARCH.md: suffixes (III, Jr), Scottish prefixes (McCloud), hyphenated names, occupation acronyms (CNC, CPA)

    Create synthetic.csv:
    - ~20 rows, 11 columns matching SOS CSV format (CONTEST NAME, COUNTY, MUNICIPALITY, CANDIDATE NAME, etc.)
    - Include edge cases: ALL CAPS names with Jr/III/McCloud, duplicate candidates in multi-county contests, municipal contests, statewide contests
    - Include columns: CONTEST NAME, COUNTY, MUNICIPALITY, CANDIDATE NAME, CANDIDATE STATUS, POLITICAL PARTY, QUALIFIED DATE, INCUMBENT, OCCUPATION, EMAIL ADDRESS, WEBSITE

    Create test_golden_files.py:
    - TestGoldenFiles class with parametrized tests:
      - test_normalize_matches_expected: normalize(before) == after
      - test_idempotency: normalize(after) == after (running on already-normalized produces no changes)
    - Use tmp_path fixture to copy before files, run normalize_file, compare against after files

    Create test_idempotency.py:
    - Use Hypothesis @given decorator with custom strategies:
      - Generate random markdown-like strings with metadata tables, candidate tables
      - st.text() for field values, st.sampled_from() for field names
    - Property: normalize_file(normalize_file(input)) == normalize_file(input)
    - Keep strategies simple -- generate valid markdown structure with random values

    Create test_uuid_handler.py:
    - TestEnsureUuid: empty ID generates UUID, valid UUID unchanged, invalid UUID raises ValueError, "--" placeholder generates UUID
    - TestRenameCandidateFile: placeholder replaced, no placeholder returns None
