**In `tests/unit/test_services/test_voter_history_service.py`:**

Add new test methods to the existing test class. These test the service-layer logic:

1. **`test_mismatch_filter_error_null_district_type`**: Create a mock election with `district_type=None`. Call `list_election_participants` with `has_district_mismatch=True`. Assert `MismatchFilterError` is raised with message containing "known district_type".

2. **`test_mismatch_filter_error_unknown_district_type`**: Create a mock election with `district_type="nonexistent_type"`. Call `list_election_participants` with `has_district_mismatch=True`. Assert `MismatchFilterError` is raised with message containing "not supported for district_type".

3. **`test_mismatch_filter_true_uses_jsonb_containment`**: Verify that when `has_district_mismatch=True` is passed with a valid election (e.g. `district_type="state_senate"`), the service:
   - Does not raise MismatchFilterError
   - Returns results (may be empty depending on mock data)
   - The query path involves `analysis_results` JOIN (can verify by checking the function returns 4-tuple with the 4th element being the district_type string)

4. **`test_mismatch_filter_false_includes_unanalyzed`**: Verify that `has_district_mismatch=False` also triggers the analysis_results JOIN path (4th return element is the district_type). Unanalyzed voters (no analysis record) must be **included** in results — the filter treats absence of analysis as non-mismatched, not as mismatched.

5. **`test_mismatch_filter_omitted_no_analysis_join`**: Verify that when `has_district_mismatch` is not specified, the 4th return element is `None` (no district_type used, no analysis_results JOIN).

6. **`test_build_mismatch_filter_true`**: Build `latest_ar = _latest_analysis_subquery()` and call `_build_mismatch_filter(latest_ar.c.mismatch_details, "state_senate", True)` and verify it returns a SQLAlchemy column element (not None, is a BinaryExpression or similar). This is a smoke test for the helper.

7. **`test_build_mismatch_filter_false`**: Build `latest_ar = _latest_analysis_subquery()` and call `_build_mismatch_filter(latest_ar.c.mismatch_details, "state_senate", False)` and verify it returns a SQLAlchemy column element.

Import `MismatchFilterError` from `voter_api.services.voter_history_service` and `_build_mismatch_filter` for direct testing.

**In `tests/integration/test_voter_history_api.py`:**

Add integration tests that hit the API endpoints via the test client:

1. **`test_participation_mismatch_null_district_type_422`**: Create an election fixture with `district_type=None`. Call `GET /elections/{id}/participation?has_district_mismatch=true`. Assert 422 status and detail message contains "known district_type".

2. **`test_participation_mismatch_unknown_district_type_422`**: Create an election fixture with `district_type="nonexistent"`. Call `GET /elections/{id}/participation?has_district_mismatch=true`. Assert 422 status and detail contains "not supported".

3. **`test_participation_mismatch_district_type_in_response`**: Create an election with `district_type="state_senate"`. Call `GET /elections/{id}/participation?has_district_mismatch=true`. Assert response contains `mismatch_district_type` field equal to `"state_senate"`.

4. **`test_participation_no_mismatch_filter_no_district_type_field`**: Call `GET /elections/{id}/participation` WITHOUT `has_district_mismatch`. Assert response contains `mismatch_district_type` as `null` (or not present / None).

5. **`test_participation_stats_mismatch_count`**: Call `GET /elections/{id}/participation/stats` for an election with a valid `district_type`. Assert response contains `mismatch_count` field (value may be 0 or null depending on test data).

6. **`test_participation_mismatch_false_422_null_district_type`**: Verify that `has_district_mismatch=false` ALSO returns 422 when `district_type` is null (not just `true`).

Follow existing test patterns in the file for fixtures, client setup, and assertion style.
